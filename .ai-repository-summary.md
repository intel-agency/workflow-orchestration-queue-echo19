# AI Repository Summary

## Repository Purpose

**OS-APOW** (Opencode-Server Agent Workflow Orchestration) is a headless agentic orchestration platform that transforms GitHub Issues into automated Execution Orders fulfilled by specialized AI agents. It shifts AI from a passive co-pilot to an autonomous background production service.

## Key Files and Their Roles

| File | Role |
|------|------|
| `pyproject.toml` | Project metadata, dependencies (FastAPI, Pydantic, HTTPX), dev tools (pytest, ruff, mypy) |
| `src/notifier_service.py` | FastAPI app: GitHub webhook ingestion, HMAC verification, event triage |
| `src/orchestrator_sentinel.py` | Async polling service: task claiming, worker lifecycle, shell bridge dispatch |
| `src/models/work_item.py` | Unified WorkItem Pydantic model with TaskType and WorkItemStatus enums |
| `src/models/github_events.py` | Pydantic schemas for GitHub webhook payloads (IssuesEvent, GitHubIssue) |
| `src/interfaces/i_task_queue.py` | ITaskQueue ABC defining provider-agnostic queue operations |
| `Dockerfile` | Python 3.12-slim with uv, installs project as editable package |
| `docker-compose.yml` | Service definitions for notifier (port 8000) and sentinel (port 8001, profile-gated) |

## Architecture Decisions

1. **Polling-First Resiliency:** Sentinel uses polling as primary discovery mechanism; webhooks are an optimization. Ensures self-healing after downtime.
2. **Provider-Agnostic Queue Interface:** `ITaskQueue` abstract base class enables future support for Linear, Notion, or custom queues.
3. **Shell-Bridge Execution:** Orchestrator interacts with worker containers exclusively via shell scripts, ensuring reproducible environments.
4. **Markdown as a Database:** GitHub Issues serve as the task queue, providing transparency, auditability, and human oversight.
5. **Python 3.12+ with uv:** Modern Python with Rust-based package manager for fast dependency resolution and container builds.

## Tech Stack

- **Python 3.12+** with FastAPI, Pydantic v2, HTTPX, Uvicorn
- **uv** for package management (Rust-based, deterministic lockfiles)
- **Docker** for worker isolation and containerized deployment
- **opencode CLI** as the AI agent runtime
- **GitHub Issues + Labels** as the task state management layer
